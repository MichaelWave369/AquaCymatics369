# AquaCymatics369 Claim Boundary

AquaCymatics369 is built to keep visual wonder and evidential honesty together.

## Claim classes

| Class | Meaning | Allowed claim | Blocked claim |
| --- | --- | --- | --- |
| Artistic | Generated visual interpretation | This render is inspired by cymatic geometry. | This proves a hidden 3D object exists. |
| Simulated | Procedural or physics-model output | This output follows the declared model assumptions. | This is measured data. |
| Inferred | Reconstruction hypothesis | This is a plausible reconstruction from an input. | The reconstruction is verification. |
| Measured | Real capture and calibration data | This result is tied to capture receipts and uncertainty notes. | Measured label without receipts. |

## v0.1 boundary

The current app uses a browser oscillator, WebGL2 shader fields, procedural harmonic layers, and claim labels. It does not yet reconstruct a physical cymatic experiment from calibrated data.

## Future measured-data receipts

Measured mode should require source media, capture conditions, frequency logs, medium and boundary description, camera/depth calibration, transformation steps, and uncertainty notes.
